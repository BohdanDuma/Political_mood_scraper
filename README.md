# YouTube Comments Sentiment Scraper & Analyzer (POLIT_SCARPER)

Інструмент для автоматичного збору коментарів з YouTube та їхнього аналізу тональності (sentiment analysis) за допомогою нейромережевих моделей. Проєкт контейнеризований через Docker і підготовлений для тривалої роботи на сервері.

📌 Коротко
* Збирає коментарі з відео/каналів YouTube.
* Аналізує тональність коментарів (позитивна / негативна / нейтральна) з використанням моделі `cardiffnlp/twitter-xlm-roberta-base-sentiment`.
* Зберігає результати у локальну SQLite базу для подальшої аналітики.
* Підтримує роботу в Docker-контейнері або роботу через системний Cron хост-машини.

🚀 Можливості
* Автоматичний скрапінг коментарів з YouTube.
* Нейромережевий аналіз тональності (Hugging Face Transformers).
* Локальна база даних SQLite для результатів та метаданих.
* Кеш моделей у `cache/huggingface` або використання глобального кешу системи — без повторного завантаження при рестарті.
* Два режими автоматизації: ізольований у Docker або нативний через Cron.

🛠 Технології
* Python 3.11 (або Miniconda)
* Hugging Face `transformers` (PyTorch)
* Pandas
* SQLite
* Docker, Docker Compose
* Linux Cron

⚠️ ВАЖЛИВЕ ЗАУВАЖЕННЯ ЩОДО DOCKER (CPU vs GPU)

За замовчуванням Docker-контейнер запускається в ізольованому середовищі й використовує ресурси процесора (CPU). Оскільки аналіз тональності виконується важкою нейромережею, робота на CPU в Docker може бути значно повільнішою, ніж нативне виконання.

* **Для базового тесту:** Ви можете запустити Docker «як є» за інструкцією нижче.
* **Для високої швидкості в Docker:** Якщо у вас є відеокарта Nvidia, вам знадобиться встановити `nvidia-container-toolkit` на хост-машину та прокинути GPU у файлі `docker-compose.yml` через блок `deploy`.
* **Найкращий локальний варіант:** Запуск напряму через Miniconda за допомогою Cron (описано нижче), що забезпечує прямий доступ до вашої відеокарти (CUDA) без додаткових налаштувань Docker.

🔧 Спосіб 1: Початок роботи через Docker (Quickstart)

1. Скопіюйте `.env.example` у `.env` та заповніть необхідні змінні оточення (API-ключі тощо).

2. Побудуйте та запустіть контейнери Docker:
```bash
   docker compose up --build -d
Запустіть збір та аналіз вручну (всередині контейнера):

Bash
   docker compose exec yt_scraper python -m src.main
Стежте за логами:

Bash
   docker compose logs -f yt_scraper
⏰ Спосіб 2: Нативний запуск через Cron (Рекомендовано для локальних ПК з GPU)

Якщо ви розгортаєте проєкт на власному комп'ютері чи ноутбуці (наприклад, Ubuntu) і хочете, щоб модель працювала максимально швидко, використовуючи вашу відеокарту та локальний кеш, налаштуйте запуск через системний cron.

Переконайтеся, що в локальному Python-середовищі (чи Miniconda) встановлені всі залежності:

Bash
   pip install -r requirements.txt
Відкрийте планувальник завдань Крон:

Bash
   crontab -e
Додайте в кінець файлу рядок для щогодинного запуску (обов'язково вкажіть ваші реальні шляхи до папки проєкту та інтерпретатора Python):

Plaintext
   0 * * * * cd /home/USER/POLIT_SCARPER && PYTHONPATH=. /home/USER/miniconda3/bin/python src/main.py >> /home/USER/POLIT_SCARPER/YT_project.log 2>&1
Параметр PYTHONPATH=. критично важливий для Крону, щоб Python правильно розпізнавав внутрішні імпорти з папки src.

⚙️ Конфігурація

Використовуйте файл .env для зберігання секретів та налаштувань (не додавайте .env у git).

Модель та її вага кешуються у cache/huggingface (директорія ігнорується в .gitignore).

База даних зберігається в data/ (також ігнорується в репозиторії).
