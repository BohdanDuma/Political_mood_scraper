# YouTube Comments Sentiment Scraper & Analyzer (POLIT_SCRAPER)

Автоматичний інструмент для збору та аналізу тональності коментарів з YouTube каналів. Проект використовує нейромережи для визначення настроїв та емоцій у текстах, допомагаючи розуміти громадські настрої та сентименти.

## 📌 Основні можливості

- **Скрапінг YouTube** — автоматичне збирання коментарів з відео та каналів
- **Аналіз тональності** — визначення позитивних, негативних та нейтральних коментарів за допомогою моделі `cardiffnlp/twitter-xlm-roberta-base-sentiment`
- **Локальна база даних** — збереження результатів у SQLite для подальшого аналізу
- **Docker-контейнер** — легко налаштовується та запускається в будь-якому середовищі
- **Кеш моделей** — вантажа моделі кешується локально для прискорення запусків
- **Два режими автоматизації** — Docker контейнер або нативний запуск через Cron на хост-машині

## 🛠 Технологічний стек

- Python 3.11 (або Miniconda)
- Hugging Face Transformers (PyTorch)
- Pandas
- SQLite
- Docker & Docker Compose
- Linux Cron (для планування)

## ⚠️ Важливо про GPU та Docker

За замовчуванням контейнер запускається на CPU. Якщо у вас є відеокарта Nvidia і ви хочете прискорити обробку:

- **Базовий тест** — запустіть Docker за інструкцією нижче (використовуватиме CPU)
- **Для GPU в Docker** — встановіть `nvidia-container-toolkit` на хост-машині і розкоментуйте відповідні параметри в `docker-compose.yml`
- **Найкраще для локального ПК** — запуск напряму через Miniconda з Cron забезпечує найкращу продуктивність при наявності GPU

## 🚀 Встановлення та запуск

### Варіант 1: Docker (Рекомендовано для серверів)

1. **Підготуйте конфігурацію**

   Скопіюйте файл `.env.example` в `.env` та заповніть API-ключі:

   ```bash
   cp .env.example .env
   ```

2. **Запустіть контейнери**

   ```bash
   docker compose up --build -d
   ```

3. **Запустіть збір та аналіз**

   ```bash
   docker compose exec yt_scraper python -m src.main
   ```

4. **Слідкуйте за логами**

   ```bash
   docker compose logs -f yt_scraper
   ```

---

### Варіант 2: Локальна установка з Cron (Рекомендовано для ПК з GPU)

Якщо ви запускаєте на власному комп'ютері, можна обійтися без Docker для кращої продуктивності.

1. **Встановіть залежності**

   Переконайтеся, що у вас встановлена Miniconda або Python 3.11:

   ```bash
   pip install -r requirements.txt
   ```

2. **Налаштуйте Cron для автоматизації**

   Відкрийте редактор Cron:

   ```bash
   crontab -e
   ```

   Додайте рядок для щогодинного запуску (змініть `/home/USER/` на свою реальну директорію):

   ```plaintext
   0 * * * * cd /home/USER/Political_mood_scraper && PYTHONPATH=. /home/USER/miniconda3/bin/python src/main.py >> /home/USER/Political_mood_scraper/scraper.log 2>&1
   ```

   **Важливо:** параметр `PYTHONPATH=.` дозволяє Python правильно імпортувати модулі з папки `src/`.

## ⚙️ Конфігурація

- **`.env` файл** — містить секрети та API-ключі (НЕ додається до репозиторію)
- **`cache/huggingface`** — локальний кеш ваг моделей (ігнорується Git)
- **`data/`** — директорія з SQLite базою та результатами

## 📂 Структура проекту

```
Political_mood_scraper/
├── cache/                      # Кеш ваг моделей
│   └── huggingface/hub/
├── data/                       # SQLite база та результати
├── src/                        # Основний код
│   ├── main.py                 # Точка входу програми
│   ├── DataTransform.py        # Обробка даних та sentiment analysis
│   ├── YouTubeLoader.py        # Логіка скрапінгу YouTube
│   └── DatabaseConnector.py    # Робота з SQLite
├── .env                        # Конфігурація (локально, не в репо)
├── Dockerfile
├── docker-compose.yml
├── pyproject.toml
└── requirements.txt
```

## 💾 Збережені дані

- **`cache/huggingface`** — кеш моделей (~1.1 GB, завантажується один раз)
- **`data/youtube_analytics.db`** — основна база даних з результатами аналізу
